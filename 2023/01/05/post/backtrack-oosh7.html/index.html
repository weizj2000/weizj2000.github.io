回溯算法(backtrack)

算法概述

回溯法也可以叫做回溯搜索法，它是一种搜索的方式。

回溯是递归的副产品，只要有递归就会有回溯。

回溯本质上是一种暴力穷举算法，把所有的可能都列举出来，所以回溯并不高效。

算法定义

回溯法解决的问题都可以抽象为树形结构（N叉树）。

因为回溯法解决的都是在集合中递归查找子集，集合的大小就构成了树的宽度，递归的深度，都构成的树的深度。

递归就要有终止条件，所以必然是一棵高度有限的树（N叉树）。

算法模版

回溯算法函数一般命名为backtracking，无返回值。

回溯三部曲：

  • 递归参数
  • 递归终止条件
  • 单层搜索逻辑
void backtracking(参数) {
    if (终止条件) {
        存放结果;
        return;
    }

    for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）) {
        处理节点;
        backtracking(路径，选择列表); // 递归
        回溯，撤销处理结果
    }
}

算法能解决的问题

回溯法，一般可以解决如下几种问题：

组合问题：N个数里面按一定规则找出k个数的集合
切割问题：一个字符串按一定规则有几种切割方式
子集问题：一个N个数的集合里有多少符合条件的子集
排列问题：N个数按一定规则全排列，有几种排列方式
棋盘问题：N皇后，解数独等等

组合问题

LeetCode77 组合

LeetCode

给定两个整数 n 和 k，返回范围 [1, n] 中所有可能的 k 个数的组合。

解题思路

如果k为2，可以使用两个for实现

var resultList = make([][2]int,0)
for i:=1;i<=n;i++ {
    for j:=i+1;j<=n;j++ {
        resultList = append(resultList, [2]{i,j})
    }
}

如果k为3，可以使用三个for实现

var resultList = make([][3]int,0)
for i:=1;i<=n;i++ {
    for j:=i+1;j<=n;j++ {
        for u:=j+1;u<=n;u++ {
        resultList = append(resultList, [3]{i,j,u})
    }
    }
}

当k为20时，需要20个for实现，显然不行，回溯法就用递归来解决嵌套层数的问题。

递归来做层叠嵌套（可以理解是开k层for循环），每一次的递归中嵌套一个for循环，那么递归就可以用于解决多层嵌套循环的问题了。

可以抽象成n叉树问题，树高为k，树宽是n，n叉树遍历。

​​

每次从集合中选取元素，可选择的范围随着选择的进行而收缩，调整可选择的范围。

每次搜索到叶子节点，就找到一个结果。

只需要把达到叶子节点的结果收集起来，就可以求得 n个数中k个数的组合集合。

结合算法模版，思路是：

终止条件：搜索到叶子节点，路径长度为k

搜索过程：for循环用来横向遍历，递归的过程是纵向遍历

剪枝：纵向递归还需要元素个数大于横向遍历剩余元素个数时，不需要横向遍历了。

Go解题

var (
    // 定义中间结果变量
    intermediateResult []int
    // 定义结果变量
    resultSet  [][]int
)

func combine(n int, k int) [][]int {
    intermediateResult, resultSet = make([]int, 0, k), make([][]int, 0) 
    backtracking(n, k, 1)
    return resultSet
}

// 递归函数，
func backtracking(n, k, startIndex int) {
    // 终止条件, 到达叶子节点，intermediateResult长度为k
    if len(intermediateResult) == k {
    tmp := make([]int, k)
    copy(tmp, intermediateResult)
        resultSet = append(resultSet,tmp)
        return
    }
    // for横向递归，startIndex记录当前横向位置
    // 集合n中至多要从该起始位置 : n - (k - path.size()) + 1，开始遍历
    for i := startIndex; i <= n - (k-len(intermediateResult)) + 1; i++ {
        // 剪枝 纵向递归还需要元素个数大于横向遍历剩余元素个数时，不需要横向遍历了
        //if k-len(intermediateResult) > n-i+1 {
        //    return
        //}
        // 处理当前节点
        intermediateResult = append(intermediateResult, i)
        // 递归，树纵向遍历
        backtracking(n, k, i + 1)
        // 回溯，撤销处理当前节点
        intermediateResult = intermediateResult[:len(intermediateResult)-1]
    }
}

C++解题

class Solution {
private:
    vector<vector<int>> resultSet;
    vector<int> intermediateResult;

    void backtracking(int n, int k, int startIndex) {
        if (intermediateResult.size() == k) {
        resultSet.push_back(intermediateResult);
        return;
    }

        for(int i = startIndex; i <= n; i++) {
            if (k-intermediateResult.size() > n-i+1 ){
                return;
            }
        intermediateResult.push_back(i);
        backtracking(n, k, i + 1);
        intermediateResult.pop_back();
    }
    }

public:
    vector<vector<int>> combine(int n, int k) {
        backtracking(n, k, 1);
        return resultSet;
    }
};

LeetCode39 组合总和

‍

给你一个无重复元素的整数数组 candidates​ 和一个目标整数 target ​​，找出 candidates ​​中可以使数字和为目标数 target​ 的所有不同组合 ，并以列表形式返回。你可以按任意顺序返回这些组合。

​candidates​ 中的同一个数字可以无限制重复被选取 。如果至少一个数字的被选数量不同，则两种组合是不同的。

对于给定的输入，保证和为 target​ 的不同组合数少于 150 个。

解题思路

数组candidates​得到所有组合，组合元素总和为target​，就是结果子集.

数组candidates​元素抽象为N叉树，递归没有层数限制，只有总和限制。

结合回溯模版，思路如下：

递归参数：

  • 定义两个变量resultSet，intermediateResult分别存放结果集和符合条件的中间结果集。
  • 定义一个sum变量统计中间结果集中的加和。也可以不定义，target​减小。

终止条件：sum等于target时，就是结果；sum大于target，递归就不必进行了。

单层搜索逻辑：

  • for循环从startIndex开始，搜索candidates​数组。
  • 因为元素可以重复选取，startIndex不递增，始终为0

剪枝：数组排序后，如果sum>target，可以不进行递归了，for循环条件可以更改为 i<candidates.size() && sum <= target

‍

Go解题

var (
    resultSet [][]int
    intermediateResult []int
)

func combinationSum(candidates []int, target int) [][]int {
    resultSet, intermediateResult = make([][]int,0), make([]int,0,len(candidates))
    sort.Ints(candidates) //剪枝
    backtracking(candidates, target, 0)
    return resultSet
}

func backtracking(candidates []int, target, indexStart int ) {
    // 终止条件
    if target == 0 {
        tmp := make([]int,len(intermediateResult))
        copy(tmp, intermediateResult)
        resultSet = append(resultSet, tmp)
        return
    }

    // 单层搜索逻辑
    for i := indexStart; i<len(candidates); i++ {
        if candidates[i] > target {
            break
        }
        intermediateResult = append(intermediateResult, candidates[i])
        // 元素可以重复用，indexStart不递增
        backtracking(candidates,target - candidates[i], i)
        intermediateResult = intermediateResult[:len(intermediateResult)-1]
    }
}

C++解题

class Solution {
private:
    vector<vector<int>> resultSet;
    vector<int> intermediateResult;

    void backtracking(vector<int>& candidates, int target, int sum, int indexStart) {
        if (sum == target) {
            resultSet.push_back(intermediateResult);
            return;
        }
        if (sum > target) {
            return;
        }

        // for (int i = indexStart; i < candidates.size(); i++) {
        for (int i = indexStart; i < candidates.size() && sum + candidates[i] <= target; i++) {
            sum += candidates[i];
            intermediateResult.push_back(candidates[i]);
            backtracking(candidates, target, sum, i);
            sum -= candidates[i];
            intermediateResult.pop_back();
        }
    }


public:
    vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
        backtracking(candidates, target, 0, 0);
        return resultSet;
    }
};

LeetCode40 组合总和II

LeetCode

给定一个候选人编号的集合 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。candidates 中的每个数字在每个组合中只能使用 一次 。注意：解集不能包含重复的组合。

解题思路

组合问题，和上面不同的在于元素不重复。

结合回溯模版，思路如下：

回溯参数：定义两个变量resultSet，intermediateResult分别存放结果集和符合条件的中间结果集。

终止条件：candidates​排序，加和等于或者大于target终止。

单层搜索逻辑：

  • for 循环从startIndex开始, 搜索candidates​数组。
  • 元素不能重复取，startIndex需要递增。
  • 组合不能重复，问题的关键在于组合去重, 使用过的元素不能重复选取。
  • 组合问题可以抽象为树形结构，那么“使用过”在这个树形结构上是有两个维度的，一个维度是同一树枝上使用过，一个维度是同一树层上使用过。
  • 元素在同一个组合内是可以重复的，但两个组合不能相同，要去重的是同一树层上的“使用过”，同一树枝上的都是一个组合里的元素，不用去重。

​

要去重的是“同一树层上的使用过”，如何判断同一树层上元素（相同的元素）是否使用过了？

树层相邻的两个元素相等，且上一个元素被使用过，那么这个元素就不需要遍历了，注定会重复的。

代码表述就是candidates[i] == candidates[i - 1]​ 且used[i - 1] == false​

如图：

​​

注意：数组必须是排序后的，used数组定义了使用状态, 当前位置使用过就为1，没有使用过就为0.

​

Go解题

var (
    // 结果集
    rs [][]int
    // 中间结果集
    irs []int
)

func combinationSum2(candidates []int, target int) [][]int {
    rs, irs = make([][]int, 0), make([]int,0,len(candidates))
    used := make([]bool, len(candidates))
    sort.Ints(candidates)
    backtracking(candidates, target, 0, used)
    return rs
}

func backtracking(candidates []int, target, indexStart int, used []bool) {
    if target == 0 {
        tmp := make([]int, len(irs))
        copy(tmp, irs)
        rs = append(rs, tmp)
        return
    }

    for i:=indexStart; i<len(candidates); i++ {
        if target < candidates[i] {
            break
        }
        if i>0 && candidates[i]==candidates[i-1] && used[i-1] == false {
            continue
        }
        irs = append(irs, candidates[i])
        used[i] = true
        backtracking(candidates, target-candidates[i], i+1, used)
        irs = irs[:len(irs)-1]
        used[i] = false
    }

}

C++解题

class Solution {
private:
    vector<vector<int>> resultSet;
    vector<int> intermediateResult;
    void backtracking(vector<int>& candidates, int target, int sum, int indexStart, vector<bool>& used) {
        if (sum > target) {
            return;
        }
        if (sum == target) {
            resultSet.push_back(intermediateResult);
            return;
        }
        for (int i = indexStart; i < candidates.size(); i++) {
            if (sum + candidates[i] > target) {
                break;
            }
            if (i>0 && candidates[i]==candidates[i-1] && used[i-1] == false) {
                continue;
            }
            sum += candidates[i];
            intermediateResult.push_back(candidates[i]);
            used[i] = true;
            backtracking(candidates, target, sum, i+1, used);
            used[i] = false;
            sum -= candidates[i];
            intermediateResult.pop_back();
        }

    }

public:
    vector<vector<int>> combinationSum2(vector<int>& candidates, int target) {
        vector<bool> used(candidates.size(), false);
        sort(candidates.begin(), candidates.end());
        backtracking(candidates, target, 0, 0, used);
        return resultSet;
    }
};
